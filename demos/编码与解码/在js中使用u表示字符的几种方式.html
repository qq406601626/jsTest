在js中，默认使用Unicode-16编码。 所以在使用\u表示字符时，如果： 1、该字符Unicode码点小于U+FFFF，那么使用\uxxxx表示。 2、该字符Unicode码点大于U+FFFF,使用Unicode-16的转换规则（详见编码表.html），把该字符转成四个字节的16进制，即使用\uxxxx(第一段)\uxxxx(第二段)。 第二种表示方式： 比较简单的方式就是直接获得该字符的Unicode码点，不经过UTF-16转换，把该码点直接转成16进制，然后使用\u{xxxxx}表示，该表示方式可以不用区分字符的Unicode码点是否大于U+FFFF。 js中几种表示字符的方式： ('z').codePointAt(0)：122。 ('z').codePointAt(0).toString(8)：172。 ('z').codePointAt(0)).toString(16)：7A。 '\172' === 'z' // true ==> 用八进制的方式表示（对应Ascii码，所以只能表达基础字符）。语法：\数字。 '\x7A' === 'z' // true ==> 用十六进制的方式表示（对应Ascii码，所以只能表达基础字符）。一个\x后面只能个一个字节，语法：\x65\x7A。 '\u007A' === 'z' // true ==> 用十六进制的方式表示。 '\u{7A}' === 'z' // true ==> 用十六进制的方式表示。 js中，在处理Unicode码点大于U+FFFF的字符时，会有几种特殊问题： 1、字符.length返回值为2。 2、for (let i = 0; i < text.length; i++){console.log(text[i]);}循环中不能正确打印出字符内容。 js中对字符串扩展了for...of循环，该循环可以正确识别大于0xFFFF的码点： 例如：for (let i of '𠮷') {console.log(i);} // 因为该循环没有使用到str.length Array.from(str)或者[...str]可以正确获得一个字符串的长度，即便该字符串中有码点大于U+FFFF的字符。 即便Array.from(str)或者[...str]可以正确获得一个字符串的长度，但是配合普通for循环使用时，还是不能正确打印字符内容。
